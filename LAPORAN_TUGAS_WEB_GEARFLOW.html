BAB I - ANALISIS SISTEM

1.1 Latar Belakang Masalah

Pengelolaan suku cadang (spare parts) dan alat kerja pada bengkel otomotif modern merupakan aspek vital yang menentukan kelancaran operasional servis. Metode pencatatan konvensional yang masih mengandalkan buku kertas atau spreadsheet sederhana memiliki kelemahan kritis, antara lain tingginya risiko human error, hilangnya bukti fisik transaksi, serta ketidakmampuan memantau stok secara real-time. Ketidakakuratan data inventaris sering kali menyebabkan kehabisan suku cadang penting di tengah proses perbaikan kendaraan, yang mengakibatkan penurunan kepuasan pelanggan.

1.2 Solusi yang Diusulkan

Untuk memecahkan permasalahan tersebut, dirancanglah GearFlow Inventory System, sebuah aplikasi berbasis web modern dengan kerangka kerja Laravel 11. Sistem ini mengintegrasikan teknologi kamera pemindai Barcode dan QR Code secara langsung melalui peramban (WebRTC Scanner), mempermudah mekanik dan staf dalam mencatat arus barang masuk dan keluar hanya dalam hitungan detik tanpa mengetik manual.

1.3 Role-Based Access Control (RBAC) & Keamanan

Sistem ini membagi hak akses menjadi dua peran utama demi keamanan data:

  • Administrator: Memiliki wewenang penuh atas kelola data master barang, manajemen mitra pemasok, penambahan akun staf baru, pemantauan log aktivitas sistem (Audit Trail), dan ekspor laporan Excel.
  • Staf Bengkel: Wewenang difokuskan pada kegiatan operasional sehari-hari, yaitu mencatat penerimaan barang masuk, transaksi pemakaian barang keluar untuk servis, dan pemindaian SKU.

BAB II - STRUKTUR DATABASE

2.1 Desain Relasi Tabel (ERD)

Database sistem dinamai gearflow_inventory_db dan dirancang menggunakan 6 entitas tabel yang saling berelasi dengan integritas referensial (Foreign Key Cascade Protection):

No Nama Tabel Fungsi / Deskripsi Utama Relasi (Foreign Key)
1 items (barangs) Menyimpan data master suku cadang, SKU barcode, stok, batas minimum, dan harga. Parent Table (Dilihat oleh transaksi)
2 suppliers (pemasoks) Menyimpan data direktori mitra distributor/pemasok suku cadang. Parent Table
3 incoming_items (barang_masuks) Mencatat riwayat penambahan stok dari pembelian/suplai. FK ke items(id) & suppliers(id)
4 outgoing_items (barang_keluars) Mencatat riwayat pemakaian suku cadang untuk servis kendaraan. FK ke items(id)
5 users (penggunas) Menyimpan akun login sistem (Admin dan Staf) dengan enkripsi bcrypt. Berelasi dengan log aktivitas
6 activity_logs Mencatat jejak audit (audit trail) setiap aksi pengguna beserta IP Address. Log independen

BAB III - IMPLEMENTASI SISTEM

3.1 Spesifikasi Lingkungan Implementasi

Sistem ini telah diimplementasikan dan diuji coba pada lingkungan pengembangan lokal dan server produksi live online dengan rincian:

  • Tautan Domain Live Production: https://www.garasifyy.site
  • Repositori GitHub: https://github.com/tech0608/gearflow-inventory
  • Framework Backend: Laravel 11.x dengan PHP-FPM versi 8.3
  • Web Server Produksi: LiteSpeed / Nginx pada Ubuntu Linux Server (Anymhost Nilou Cloud)
  • Keamanan HTTPS: Sertifikat Let's Encrypt AutoSSL 256-bit Encryption

3.2 Implementasi Fitur Unggulan

Beberapa inovasi fitur yang diimplementasikan pada GearFlow Inventory System mencakup:

  1. Html5-QRCode Camera Scanner: Integrasi pustaka pemindai optik berbasis JavaScript yang memungkinkan kamera peramban membaca kode batang (Barcode) maupun QR Code suku cadang secara real-time.
  2. Algoritma Proteksi Stok Minus: Validasi tingkat kontroler mencegah pengurangan stok jika jumlah barang keluar yang diinput melebihi sisa stok yang tersedia di database.
  3. Dynamic Chart Dashboard: Visualisasi statistik transaksi 7 hari terakhir menggunakan Chart.js dengan perlindungan anti-XSS melalui direktif Blade @json().

BAB V - KESIMPULAN DAN SARAN

5.1 Kesimpulan

Berdasarkan hasil perancangan, implementasi, dan pengujian server produksi yang telah dilakukan pada proyek GearFlow Inventory System, dapat disimpulkan bahwa:

  1. Sistem manajemen inventaris berbasis Laravel 11 ini mampu meningkatkan efisiensi dan keakuratan pencatatan stok bengkel otomotif secara signifikan dibandingkan metode manual.
  2. Integrasi fitur pemindai Barcode/QR Code melalui kamera peramban terbukti mempercepat proses pencarian barang dan pencatatan transaksi masuk/keluar hingga 80% lebih cepat tanpa memerlukan perangkat lunak tambahan.
  3. Penerapan arsitektur keamanan Role-Based Access Control (RBAC) dan Activity Logging mampu menjaga integritas data serta mencegah manipulasi stok oleh pihak yang tidak berwenang.
  4. Aplikasi telah berhasil di-deploy pada server produksi secara live (https://www.garasifyy.site) dengan performa tinggi dan perlindungan SSL HTTPS berstandar industri.

5.2 Saran

Untuk pengembangan sistem di masa yang akan datang, disarankan untuk menambahkan modul Point of Sales (POS / Kasir) terintegrasi agar biaya jasa servis mekanik dan harga suku cadang dapat langsung dicetak menjadi struk faktur untuk pelanggan bengkel.
